# Plan de deploiement de la documentation du code

Objectif: appliquer progressivement `CODE_DOCUMENTATION_CONVENTION.md` au code TypeScript maintenu a la main et au CSS du portage.

Ce plan part de l'inventaire reel du repo effectue le 2026-06-05.

## Regles d'application

- [ ] Les fichiers TypeScript maintenus a la main doivent recevoir une entete et une documentation de toutes les entites.
- [ ] Le CSS maintenu a la main doit recevoir des sections commentees.
- [ ] Les fichiers generes, JSON, Markdown, PNG, BIN, logs et configs non TS sont hors perimetre de cette campagne.

## Phase 0 - Hygiene avant documentation

- [ ] Decider quoi faire de `.codex-vite.log`.
- [ ] Decider quoi faire de `.codex-vite.err.log`.
- [ ] Verifier que les logs temporaires ne doivent pas rester versionnes.
- [ ] Ajouter une note `.gitignore` si ces logs doivent etre ignores.
- [ ] Ne pas documenter les logs.
